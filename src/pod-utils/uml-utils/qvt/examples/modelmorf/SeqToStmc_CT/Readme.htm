Mapping between sequence diagrams and state machines (Demonstrates Sequence Patterns)

----------------------------------------------------------------------------------------------------------------------

 

Folder ‘spec’ contains :

   SeqMM.xml : Sequence Diagram Meta Model

   StmcMM.xml : State Machine Meta Model

   SeqUM.xml : A sample sequence diagram model

   StmcUM.xml : A sample state machine Model

   SeqToStmc.qvt : QVT specification file for sequence diagram to state machine mapping.

 

 

Sequence Diagram Meta Model :

A Sequence Diagram has a set of lifelines representing objects that exchange messages. A lifeline has events corresponding to messages.

The events occurring on a lifeline are ordered in time. This can be modeled by creating an ordered association between 'LifeLine' and 'MessageEvent', where 'MessageEvent' extends 'Event'.

 

See figure 'Sequence Diagram Meta Model for example SeqToStm_CT' in file SeqToStmc_CT.pdf

 

 

State Machine Meta Model :

The state machine meta model is straight-forward - there are states and transitions.

 

See figure 'State Machine Meta Model for example SeqToStm_CT' in file SeqToStmc_CT.pdf

 

 

The example specifies a mapping between sequence diagrams and state machines.

 

The behavior of a class can be specified by a state machine. A lifeline is consistent with the state machine of the class of the object of the lifeline if following condition holds : corresponding to the event sequence of the lifeline, there exists a transition path in the state machine where the events occur in the same relative order.

 

 

This example demonstrates the use of Collection (Sequence) Patterns in QVT specification file.

 

 

The sample Sequence Diagram model is given in file SeqUM.xml file contains:

  • Class :
    • ElevatorController : Name='ElevatorController'; object={ElevatorController}
    • Elevator : Name='Elevator'; object={Elevator}
  • Object :
    • ElevatorController : Name='ElevatorController'; lifeline={ElevatorController}
    • Elevator : Name='Elevator'; lifeline={Elevator}
  • LifeLine :
    • ElevatorController : Name='ElevatorController'; object={ElevatorController}; event={SendOpenDoor, SendCloseDoor, SendLockDoor}
    • Elevator : Name='Elevator'; object={Elevator};event={ReceiveOpenDoor, ReceiveCloseDoor, ReceiveLockDoor}
  • MessageEvent :
    • SendOpenDoor : Name='SendOpenDoor'; type=Send
    • SendCloseDoor : Name='SendCloseDoor'; type=Send
    • SendLockDoor : Name='SendLockDoor'; tyep=Send
    • ReceiveOpenDoor : Name='ReceiveOpenDoor'; type=Receive
    • ReceiveCloseDoor : Name='ReceiveCloseDoor'; type=Receive
    • ReceiveLockDoor : Name='ReceiveLockDoor'; tyep=Receive

 

See figure 'Sequence Diagram model for example SeqToStm_CT' in file SeqToStmc_CT.pdf.

 

 

The sample State Machine model is given in file StmcUM.xml file contains:

  • Class :
    • Elevator : Name='Elevator'; stateMachine={Elevator}
    • ElevatorController : Name='ElevatorController'; stateMachine={ElevatorController}
  • StateMachine :
    • Elevator : Name='Elevator'; owner={Elevator}; state={START, DoorOpened, DoorClosed, DoorLocked}; transition={ReceiveOpenDoor, ReceiveCloseDoor, ReceiveLockDoor, ReceiveOpenDoor}
    • ElevatorController : Name='ElevatorController'; owner={ElevatorController}; state={START, DoorOpened, DoorClosed, DoorLocked}; transition={SendOpenDoor, SendCloseDoor, SendLockDoor, SendOpenDoor}
  • State :
    • START : Name='START';owner={Elevator}; inTransition={}; outTransition={ReceiveOpenDoor}
    • DoorOpened : Name='DoorOpened'; owner={Elevator}; inTransition={ReceiveOpenDoor, ReceiveOpenDoor}; outTransition={ReceiveCloseDoor}
    • DoorClosed : Name='DoorClosed'; owner={Elevator}; inTransition={ReceiveCloseDoor}; outTransition={ReceiveLockDoor, ReceiveOpenDoor}
    • DoorLocked : Name='DoorLocked'; owner={Elevator}; inTransition={ReceiveLockDoor}; outTransition={}
    • START : Name='START'; owner={ElevatorController}; inTransition={}; outTransition={SendOpenDoor}
    • DoorOpened : Name='DoorOpened'; owner={ElevatorController}; inTransition={SendOpenDoor, SendOpenDoor};  outTransition={SendCloseDoor}
    • DoorClosed : Name='DoorClosed'; owner={ElevatorController}; inTransition={SendCloseDoor}; outTransition={SendLockDoor, SendOpenDoor}
    • DoorLocked : Name='DoorLocked'; owner={ElevatorController};inTransition={SendLockDoor}; outTransition={}
  • Transition :
    • SendOpenDoor : Name='SendOpenDoor'; owner={ElevatorController}; fromState={START}; toState={DoorOpened}
    • SendCloseDoor : Name='SendCloseDoor'; owner={ElevatorController}; fromState={DoorOpened};  toState={DoorClosed}
    • SendLockDoor : Name='SendLockDoor'; owner={ElevatorController}; fromState={DoorClosed}; toState={DoorLocked}
    • SendOpenDoor : Name='SendOpenDoor'; owner={ElevatorController}; fromState={DoorClosed}; toState={DoorOpened}
    • ReceiveOpenDoor : Name='ReceiveOpenDoor'; fromState={START}; toState={DoorOpened}
    • ReceiveCloseDoor : Name='ReceiveCloseDoor'; fromState={DoorOpened}; toState={DoorClosed}
    • ReceiveLockDoor : Name='ReceiveLockDoor'; fromState={DoorClosed}; toState={DoorLocked}
    • ReceiveOpenDoor : Name='ReceiveOpenDoor'; fromState={DoorClosed}; toState={DoorOpened}
  • Event :
    • SendOpenDoor : Name='SendOpenDoor' (correspondent to the Transition from START State to DoorOpened State)
    • SendCloseDoor : Name='SendCloseDoor'
    • SendLockDoor : Name='SendLockDoor'
    • SendOpenDoor : Name='SendOpenDoor' (correspondent to the Transition from DoorClosed State to DoorOpened State)
    • ReceiveOpenDoor : Name='ReceiveOpenDoor' (correspondent to the Transition from START State to DoorOpened State)
    • ReceiveCloseDoor : Name='ReceiveCloseDoor'
    • ReceiveLockDoor : Name='ReceiveLockDoor'
    • ReceiveOpenDoor : Name='ReceiveOpenDoor' (correspondent to the Transition from DoorClosed State to DoorOpened State)

 

See figure 'State Machine model for example SeqToStm_CT' in file SeqToStmc_CT.pdf.

 

 

Example QVT file (SeqToStmc.qvt) is run in checkonly mode in stmc direction. It checks whether the sequence of events of a lifeline in a Sequence Diagram model is a subsequence in the sequence of states in State Machine model. Here the models are consistent.

 

 

To run this example execute the batch file ‘run.bat’ for windows or  the shell script ‘run.sh’ for unix.
