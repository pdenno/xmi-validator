Transforming an UML model into a RDBMS model (Demonstrates 'default_values' and 'opposite' feature)

------------------------------------------------------------------------------------------------------------------------------------

 

This example demonstrates the use of 'default_values' and 'opposite' constructs in qvt specification file.

 

'default_values' feature : This feature is used to specify a default value for a target domain variable when it does not get a binding from the evaluation of other relation expressions. This feature is relevant only for enforcement.

 

'opposite' feature : 'opposite' is used to specify the non-navigable end of an association. e.g. UML Meta model has an association "attribute" from Class to Attribute, which is non-navigable in Class direction. In this case, opposite can be used as "opposite(umlmm::Class.attribute)" to navigate in Class direction.

Syntax :

"opposite" "(" <fully-qualified-class-pathname> "." <navigable-end> ")"

 

 

Folder ‘spec’ contains :

   UmlToRel.qvt : QVT specification file for UmlToRel transformation.

 

Folder ‘spec/Example1’ contains :

   UmlMM.xml : UML Meta Model

   RelMM.xml : RDBMS Meta Model

   UmlUM1.xml : A sample UML model        

 

Folder ‘spec/Example2’ contains :

   UmlMM.xml : UML Meta Model

   RelMM.xml : RDBMS Meta Model

   RelUM2.xml : A sample RDBMS model     

 

 

UML metamodel:

See figure 'Simple UML Meta Model for example UmlToRel' in file UmlToRel.pdf file.

 

 

RDBMS metamodel:

See figure 'Simple RDBMS Meta Model for example UmlToRel' in file UmlToRel.pdf file.

 

 

This example maps classes of a simple UML model to tables of a simple RDBMS model & vice versa.

  • A class maps to a table.
  • Attributes of the class map to columns of the table.

 

 

Example1

----------

 

The sample UML model is given in file UmlUM1.xml file contains:

  • Class :
    • Employee : name='Employee' ; attribute={Name, EmployeeNo, IsPermanent}
  • Attribute :
    • Name : name='Name' ; type='String'
    • EmployeeNo : name='EmployeeNo' ; type='Int'
    • IsPermanent : name='IsPermanent' ; type='Binary'

 

 

Example QVT file (UmlToRel.qvt) is run in enforce mode in rdbms direction. It enforces RDBMS model & exports it in RelUM1.xml file.

 

 

To run this example execute the batch file ‘run1.bat’ for windows or  the shell script ‘run1.sh’ for unix.

 

 

Example2

----------

 

The sample RDBMS model is given in file RelUM2.xml file contains:

  • Table :
    • Company : name='Company' ; column={Name, NoOfEmployees, IsMNC}
  • Column :
    • Name : name='Name' ; dataType='Varchar'
    • NoOfEmployees : name='NoOfEmployees' ; dataType='Number'
    • IsMNC : name='IsMNC' ; dataType='Raw'

 

 

Example QVT file (UmlToRel.qvt) is run in enforce mode in uml direction. It enforces UML model & exports it in UmlUM2.xml file.

 

 

To run this example execute the batch file ‘run2.bat’ for windows or  the shell script ‘run2.sh’ for unix.
