Transforming a UML model that uses multiple inheritance into a Java model that uses single inheritance and interfaces

--------------------------------------------------------------------------------------------------------------------------------------

 

Folder ‘spec’ contains:

umlMM.xml : UML Meta Model

javaMM.xml : JAVA Meta Model

transportuml.xml : A sample UML Model

          MiToSi.qvt : QVT specification file for the transformation.

 

 

UML metamodel:

It consists of a Class 'Class' with properties 'name' and 'type', and an association 'supers' from 'Class' to 'Class'.

See figure 'Simple UML Meta Model for example MiToSi' in file MiToSi.pdf.

 

 

JAVA metamodel:

It contains two Classes 'Class' and 'Interface' each having a property 'name', an association 'extends' from 'Class' to 'Class', an association 'implements' from 'Class' to 'Interface', and an association 'extends' from 'Interface' to 'Interface'.

See figure 'Simple Java Meta Model for example MiToSi' in file MiToSi.pdf.

 

 

Transformation MiToSi transforms a UML class model to a JAVA class model, converting multiple inheritance to single inheritance. The idea is that only the class hierarchy that specifies elements of an application domain (as indicated by a root class being tagged 'DomainRoot') is retained in the java model; non-domain classes are transformed to interfaces.

It uses the following mapping rules:

  • An UML Class maps to a Java Class if its type is "DomainRoot" or if any of its super classes maps to a Java Class.
  • An UML Class, which does not inherit from a class whose type is “DomainRoot”, is mapped to a Java Interface.
  • An inheritance association between two UML classes maps to either an 'Extends' or an 'Implements' association between the corresponding Java classes and/or interfaces

 

 

The sample UML model given in file transportuml.xml contains:

  • Uml Class :
    • Engine : subs={SteamEngine, ICEngine}
    • SteamEngine : supers={Engine}
    • ICEngine : supers={Engine}; subs={TwoStrokeEngine, FourStrokeEngine}
    • TwoStrokeEngine : supers={ICEngine}; subs={TwoWheeler, ThreeWheeler}
    • FourStrokeEngine : supers={ICEngine}; subs={FourWheeler}
    • Vehicle : type="DomainRoot"; subs={FourWheeler, ThreeWheeler, TwoWheeler}
    • FourWheeler : supers={Vehicle, FourStrokeEngine}
    • TwoWheeler: supers={Vehicle, TwoStrokeEngine}
    • ThreeWheeler : supers={Vehicle, TwoStrokeEngine}

 

 

To run this example execute the batch file ‘run.bat’ for windows or  the shell script ‘run.sh’ for unix.

 

 

The generated java model transportjava.xml will contain:

  • Java Class
    • Vehicle
    • FourWheeler implements FourStrokeEngine
    • ThreeWheeler implements TwoStrokeEngine
    • TwoWheeler implements TwoStrokeEngine
  • Java Interface
    • Engine
    • ICEngine
    • SteamEngine
    • FourStrokeEngine
    • TwoStrokeEngine

 

 

Trace model output :

The command line usage in run.bat/run.sh has the option :

          -of trace.xml

This will output the trace model, created during transformation execution, to file ‘trace.xml’ in XMI format. The meta model for this is given in XMI format in file extJar/TraceMetaModel.xml.
