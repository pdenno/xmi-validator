Copying all operations of an abstract class to its concrete sub classes (Demonstrates 'in-place transformation')

-------------------------------------------------------------------------------------------------------------------------------------------

 

Folder ‘spec’ contains:

ClassMM_IN.xml : UML meta model

ClassUM_IN.xml : A sample UML model

AbstractToConcrete.qvt : QVT specification file to copy operations of an abstract class to its concrete sub classes

 

 

UML metamodel:

  • It has Class & Operation.
  • Class has property ‘isAbstract’ to indicate whether it is an abstract or a concrete class.
  • Operation has parameters.
  • Parameter has Type
  • Type is the supertype of Class and PrimitiveDataType.

 

See figure 'Meta Model for UML for example AbstractToConcrete' in file AbstractToConcrete.pdf.

 

 

The sample UML model ClassUM_IN.xml contains:

  • An abstract class ‘Instrument’ has operation ‘play’
    • Operation ‘play’ has parameters ‘pitchAlphabet’ & ‘octaveFrequency’
      • Parameter ‘pitchAlphabet’ has type ‘character’
      • Parameter ‘octaveFrequency’ has type ‘integer’
  • Abstract class ‘Instrument’ is inherited by concrete classes ‘Percussion’, ‘Stringed’, ‘Brass’ & ‘Wind’

 

 

Command line usage for in-place:

The transformation has signature as

          transformation AbstractToConcrete(uml1:UMLMM; uml2:UMLMM)

‘uml2’ uses same model as ‘uml1’. This can be given as

          -u uml1 -f ClassUM_IN.xml -u uml2 -ui uml1

 

 

To run this example execute the batch file ‘run.bat’ for windows or  the shell script ‘run.sh’ for unix.

 

 

The updated UML model ClassUM_IN.xml is expected to contain:

  • An abstract class ‘Instrument’ has operation ‘play’
    • Operation ‘play’ has parameters ‘pitchAlphabet’ & ‘octaveFrequency’
      • Parameter ‘pitchAlphabet’ has type ‘character’
      • Parameter ‘octaveFrequency’ has type ‘integer’
  • Abstract class ‘Instrument’ is inherited by concrete classes ‘Percussion’, ‘Stringed’, ‘Brass’ & ‘Wind’
  • Classes ‘Percussion’, ‘Stringed’, ‘Brass’ & ‘Wind’ has operation ‘play’ and corresponding parameters ‘pitchAlphabet’ & ‘octaveFrequency’
