Transforming  an UML model into a RDBMS model (Demonstrates 'import' feature)

--------------------------------------------------------------------------------------------------------

 

This example demonstrates the use of 'import' feature in QVT specification file.

 

'import' feature : 'import' feature imports the transformation specifications that are specified in other QVT files. It is needed to map the source and target models using the relations that are specified in the transformations of imported QVT files.

 

 

Folder ‘spec’ contains :

umlMM.xml : UML Meta Model

rdbmsMM.xml : RDBMS Meta Model

umlUM.xml : A sample UML model

rdbmsUM.xml : A sample RDBMS model

UmlToRdbms_1.qvt : QVT specification file for UmlToRdbms transformation.

UmlToRdbms_2.qvt : QVT specification file for UmlToRdbms transformation which will be imported by UmlToRdbms_1.qvt.

 

 

UML metamodel:

UML metamodel contains Package & Class. The Class is contained in the Package.

See figure 'Simple UML Meta Model for example Import' in file Import.pdf.

 

 

RDBMS metamodel:

RDBMS metamodel contains Schema & Table. The Table is contained in the Schema.

See figure 'Simple RDBMS Meta Model for example Import' in file Import.pdf.

 

 

This example maps Packages and their corresponding Classes in UML model to Schemas & their corresponding Tables in RDBMS model.

 

 

The sample UML model is given in file umlUM.xml contains:

  • Package :
    • Company : name='Company' ; hasClass={Employee, Employer}
  • Class :
    • Employee : name='Employee'
    • Employer : name='Employer'

 

 

The sample RDBMS model is given in file rdbmsUM.xml contains:

  • Schema :
    • Company : name='Company' ; hasTable={Employee, Employer}
  • Table :
    • Employee : name='Employee'
    • Employer : name='Employer'

 

 

Example QVT file (UmlToRdbms_1.qvt) is run in checkonly mode in rdbms direction. It checks the consistency between the UML model and RDBMS model.

 

 

To run this example execute the batch file ‘run.bat’ for windows or  the shell script ‘run.sh’ for unix.

 

 

NOTE : To run the specification containing 'import' successfully, the location of the imported files must be specified to 'QVT_IMPORT_PATH' field in qvt.cfg file. A sample qvt.cfg file is given with this example. Copy it to tool folder before running this example.
