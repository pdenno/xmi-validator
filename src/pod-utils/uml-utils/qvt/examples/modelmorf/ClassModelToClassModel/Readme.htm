Mapping two object models

----------------------------------

 

This example checks if two object models are equivalent. It considers them equivalent if for each class in one model there exists a corresponding class in the other model with the same name and if the inheritance hierarchies of the two classes are the same. This example demonstrates the use of set pattern.

 

 

Folder ‘spec’ contains :

          ClassModelToClassModel.qvt : QVT specification file for mapping object models.

 

Folder ‘spec/Example1’ contains :

          ClassMM.xml : CLASS Meta Model

          ClassUM1.xml : A sample class model

          ClassUM2.xml : A sample class model

 

Folder ‘spec/Example2’ contains :

          ClassMM.xml : CLASS Meta Model

          ClassUM3.xml : A sample class model

          ClassUM4.xml : A sample class model

 

 

CLASS metamodel:

See figure 'Simple CLASS Meta Model for example ClassModelToClassModel' in file ClassModelToClassModel.pdf

 

 

Example1

----------

 

The sample source model is given in file ClassUM1.xml contains:

  • Class :
    • Polygon : name='Polygon'; subs={Quadrilateral ,Triangle}; supers={}
    • Quadrilateral : name='Quadrilateral'; subs={Rhombus ,Rectangle}; supers={Polygon}
    • Triangle : name='Triangle'; subs={IsoscelesTriangle ,ScaleneTriangle}; supers={Polygon}
    • Rhombus : name='Rhombus'; subs={Square}; supers={Quadrilateral}
    • Rectangle : name='Rectangle'; subs={Square}; supers={Quadrilateral}
    • Square : name='Square'; subs={}; supers={Rhombus ,Rectangle}
    • IsoscelesTriangle : name='IsoscelesTriangle'; subs={EquilateralTriangle}; supers={Triangle}
    • ScaleneTriangle : name='ScaleneTriangle'; subs={}; supers={Triangle}
    • EquilateralTriangle : name='EquilateralTriangle'; subs={}; supers={IsoscelesTriangle}

    

 

The sample target model is given in file ClassUM2.xml contains:

  • Class :
    • Polygon : name='Polygon'; subs={Quadrilateral ,Triangle}; supers={}
    • Quadrilateral : name='Quadrilateral'; subs={Rhombus ,Rectangle}; supers={Polygon}
    • Triangle : name='Triangle'; subs={IsoscelesTriangle ,ScaleneTriangle}; supers={Polygon}
    • Rhombus : name='Rhombus'; subs={Square}; supers={Quadrilateral}
    • Rectangle : name='Rectangle'; subs={Square}; supers={Quadrilateral}
    • Square : name='Square'; subs={}; supers={Rhombus ,Rectangle}
    • IsoscelesTriangle : name='IsoscelesTriangle'; subs={EquilateralTriangle}; supers={Triangle}
    • ScaleneTriangle : name='ScaleneTriangle'; subs={}; supers={Triangle}
    • EquilateralTriangle : name='EquilateralTriangle'; subs={}; supers={IsoscelesTriangle}

    

 

Here the source and target models are consistent with each other.

 

 

Example QVT file (ClassModelToClassModel.qvt) is run in checkonly mode in uml2 direction. The transformation checks the consistency between uml1 and uml2 models. Here the models are consistent.

 

 

To run this example execute the batch file ‘run1.bat’ for windows or  the shell script ‘run1.sh’ for unix.
