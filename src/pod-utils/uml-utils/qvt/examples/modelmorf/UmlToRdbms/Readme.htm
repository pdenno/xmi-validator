Transforming a UML model into an RDBMS model

--------------------------------------------------------------

 

Folder ‘spec’ contains :

umlMM.xml : UML Meta Model

rdbmsMM.xml : RDBMS Meta Model

umlUM.xml : A sample UML model

rdbmsUM.xml : A sample RDBMS model

UmlToRdbms.qvt : QVT specification file for UmlToRdbms transformation.

 

 

UML metamodel:

See figure 'Simple UML Meta Model for example UmlToRdbms' in file UmlToRdbms.pdf file.

 

 

RDBMS metamodel:

See figure 'Simple RDBMS Meta Model for example UmlToRdbms' in file UmlToRdbms.pdf file.

 

 

This example maps persistent classes of a simple UML model to tables of a simple RDBMS model.

  • A persistent class maps to a table, a primary key and an identifying column.
  • Attributes of the persistent class map to columns of the table
    • An attribute of a primitive datatype maps to a single column
    • An attribute of a complex data type maps to a set of columns corresponding to its exploded set of primitive datatype attributes
    • Attributes inherited from the class hierarchy are also mapped to the columns of the table.
  • An association between two persistent classes maps to a foreign key relationship between the corresponding tables.

 

 

The sample UML model in file umlUM.xml contains:

  • Package :
    • Company
  • Class :
    • Employer : kind='Persistent'; attribute={Age}
    • Employee : kind='Persistent'; attribute={Info}
    • Person : attribute={Name}
    • EmployeeInfo : attribute={EmployeeNo}
  • Attribute :
    • Name : type=PrimitiveDataType STRING
    • Age : type=PrimitiveDataType INTEGER
    • Info : type=Class EmployeeInfo
    • EmployeeNo : type=PrimitiveDataType INTEGER
  • Association :
    • employs : source={Employer}; destination={Employee}

 

 

The sample RDBMS model in file rdbmsUM.xml contains:

  • Schema :
    • Company
  • Table :
    • Employer : column={Employer_tid, Age, Name}; hasKey={Employer_pk}
    • Employee : column={Employee_tid, Name, _Info_EmployeeNo}; hasKey={Employer_pk}

 

 

If the example is run in checkonly mode following inconsistency will be reported:

Association ‘employs’ between Class ‘Employer’ & ‘Employee’ in Package ‘Company’ does not find matching ForeignKey in Schema.

 

To run this example in checkonly mode execute the batch file ‘run1.bat’ for windows or  the shell script ‘run1.sh’ for unix.

 

 

If the example is run in enforce mode, enforcement removes the inconsistency by creating a ForeignKey ‘Employer_employes_Employee’ in Schema ‘Company’ & Table ‘Employer’ and association ‘refersTo’ with Key ‘Employee_pk’ of Table ‘Employee’. Also a Column ‘Employer_employes_Employee_tid’ is created and associated to Table ‘Employer’.

 

To run this example in enforce mode execute the batch file ‘run2.bat’ for windows or  the shell script ‘run2.sh’ for unix.
