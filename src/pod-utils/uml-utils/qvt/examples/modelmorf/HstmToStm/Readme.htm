Transforming a hierarchical state machine into a flat state machine

-----------------------------------------------------------------------------------

 

Folder ‘spec’ contains:

hstmMM.xml : Hierarchical state machine meta model

stmMM.xml : Flat state machine meta model

hstmUM.xml : A sample hierarchical state machine model

hstmtostm.qvt : QVT specification file to transform a hierarchical state machine into a flat state machine

 

 

Hierarchical state machine metamodel:

It consists of states & transitions. A state may contain other states.

See figure 'Meta Model for Hierarchical State Machine for example HstmToStm' in file HstmToStm.pdf.

 

 

Flat state machine metamodel:

It consists of states & transitions. A state cannot contain another state.

See figure 'Meta Model for Flat State Machine for example HstmToStm' in file HstmToStm.pdf.

 

 

The transformation HstmToStm transforms a hierarchical state machine model named hstm into a flat state machine model named stm, as per the following mappings:

  • A leaf state in hstm is mapped to a state in stm.
  • A transition t between States s and s' in hstm is mapped to:
    • A Transition t in stm if both States s and s' are leaf states.
    • If State s is a leaf State and State s' is a non leaf State then Transitions are created in stm from the state which is mapped to s, to all those States in stm which are mapped to leaf states contained in s'.
    • If State s is a non leaf State and State s' is leaf State then Transitions are created in stm from all the states in stm which are mapped to the leaf states contained in s, to the State which is mapped to s' in stm.
    • If both States s and s' are non leaf States then Transitions are created in stm from all the states in stm which are mapped to the leaf states contained in s, to all those States which are mapped to leaf states contained in s'.

 

 

The sample hierarchical state machine model hstmUM.xml contains:

  • State : s1, s2, s3, s4, s5, s6, s7, s8, s9, s10, s11, s12, s13 & s14.
    • Non-leaf States :  
      • s1 contains s3 & s4
      • s2 contains s5 & s6
      • s3 contains s7 & s8
      • s4 contains s9 & s10
      • s5 contains s11 & s12
      • s6 contains s13 & s14
    • Leaf States : s7, s8, s9, s10, s11, s12, s13 & s14.
  • Transition :
    • t1_2   between s1 & s2
    • t3_4   between s3 & s4
    • t5_6   between s5 & s6
    • t7_8   between s7 & s8
    • t9_10  between s9 & s10
    • t11_12 between s11 & s12
    • t13_14 between s13 & s14

 

 

To run this example execute the batch file ‘run.bat’ for windows or  the shell script ‘run.sh’ for unix.

 

 

The generated flat state machine model stmUM.xml is expected to contain:

  • State : s7, s8, s9, s10, s11, s12, s13 & s14.
  • Transition :
    • t7_8  between  s7 & s8.
    • t9_10  between  s9 & s10.
    • t11_12  between  s11 & s12.
    • t13_14  between  s13 & s14.
    • t3_4  between  s7 & s9, s7 & s10, s8 & s9, s8 & s10.
    • t5_6  between  s11 & s13, s11 & s14, s12 & s13, s12 & s14.  
    • t1_2  between  s7 & s11, s7 & s12, s7 & s13, s7 & s14, s8 & s11, s8 & s12, s8 & s13, s8 & s14, s9 & s11, s9 & s12, s9 & s13, s9 & s14, s10 & s11, s10 & s12, s10 & s13, s10 & s14.
